# Phase 09 — RSS Feed

## Objective

Generate two static RSS 2.0 feeds (`/feed.xml` in pt-BR and `/feed-en.xml` in en-US) from the 50 most recent validated/curated articles. Add autodiscovery `<link>` tags to `index.html`. The feeds are generated by `generate_rss_feed.py` and committed to `site/public/` by the `collect.yml` workflow.

## Input Context

- `docs/prompt-eleicoes2026-v5.md` lines 562-604 — `generate_rss_feed.py` full implementation spec
- `data/articles.json` — Articles with summaries (from Phase 06)
- `site/index.html` — App shell (from Phase 04)
- `.github/workflows/collect.yml` — Already calls `generate_rss_feed.py` as a stub (Phase 05)

## Deliverables

### 1. `scripts/generate_rss_feed.py`

Replaces the Phase 05 stub.

**Key behaviors:**
- Read `data/articles.json`
- Filter: articles with `status` in `["validated", "curated"]` only
- Sort by `published_at` descending, take first 50
- Generate `site/public/feed.xml` (pt-BR) using `xml.etree.ElementTree`
- Generate `site/public/feed-en.xml` (en-US)
- RSS 2.0 structure per spec:
  ```xml
  <rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
    <channel>
      <title>Eleições BR 2026</title>
      <link>https://eleicoes2026.com.br</link>
      <description>Monitoramento em tempo real...</description>
      <language>pt-BR</language>
      <atom:link href="https://eleicoes2026.com.br/feed.xml" rel="self" type="application/rss+xml"/>
      <item>
        <title>...</title>
        <link>...</link>
        <description><!-- summaries["pt-BR"] --></description>
        <pubDate>...</pubDate>
        <guid><!-- article.id --></guid>
        <category><!-- each candidate_mentioned --></category>
      </item>
    </channel>
  </rss>
  ```
- Write XML declaration: `encoding="unicode"` with `xml_declaration=True`
- Create `site/public/` directory if it doesn't exist
- Print summary: "Generated feed.xml (X items) and feed-en.xml (X items)"
- **Idempotent:** running twice overwrites with identical content

**Fallback for missing summary:** if `summaries["pt-BR"]` is missing, use `article["title"]` as `<description>`.

**`pubDate` format:** RFC 2822 (e.g. `Mon, 05 Mar 2026 10:00:00 +0000`). Use `email.utils.formatdate` from stdlib:
```python
from email.utils import formatdate
from datetime import datetime
dt = datetime.fromisoformat(article["published_at"].replace("Z", "+00:00"))
pub_date = formatdate(dt.timestamp(), usegmt=True)
```

### 2. `site/index.html` — RSS autodiscovery

Add inside `<head>`:
```html
<link rel="alternate" type="application/rss+xml" title="Eleições BR 2026" href="/feed.xml" />
<link rel="alternate" type="application/rss+xml" title="Brazil Elections 2026" href="/feed-en.xml" />
```

### 4. Unit tests — `scripts/test_generate_rss_feed.py`

- `test_feed_xml_is_valid_rss` — output parses as valid XML with `<rss>` root and `version="2.0"`
- `test_feed_limited_to_50_items` — more than 50 articles produces exactly 50 `<item>` elements
- `test_feed_pt_uses_pt_summaries` — `<description>` in `feed.xml` uses `summaries["pt-BR"]`
- `test_feed_en_uses_en_summaries` — `<description>` in `feed-en.xml` uses `summaries["en-US"]`
- `test_feed_skips_raw_articles` — articles with `status: "raw"` are excluded
- `test_feed_pubdate_is_rfc2822` — `<pubDate>` is valid RFC 2822 format
- `test_idempotent_double_run` — running twice produces byte-identical output

## Constraints

- Use only Python stdlib (`xml.etree.ElementTree`, `email.utils`, `datetime`, `pathlib`) — no external XML libraries
- The RSS feed must validate against the W3C RSS 2.0 specification (use the test suite to verify)
- Only `validated` and `curated` articles appear in the feed — `raw` articles are excluded
- The feed URL in `<atom:link>` must match the production domain `https://eleicoes2026.com.br` (hardcoded, not configurable — this is a static site)
- `site/public/` is the correct output directory for static assets that bypass Vite processing
